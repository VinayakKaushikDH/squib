squib.glass

swift handoff · frosted
Frozen reference for the Swift port. One glass treatment — Frosted. Every surface (permission · plan · elicitation · chat) lives in this file as ready-to-paste HTML, with the full portable CSS in a single block at the top. Class names use the .sq-* prefix so they won't collide with whatever wrapper you bring in. JS is optional — only the chat tool-call rows need a click handler.

05 JS bridge

Wiring it to Swift.

Minimum viable bridge — Swift posts payloads in, the page returns user actions out. Same shape as your existing BubbleWindow.swift.

bridge contractwindow.squib
// ===== inbound (Swift → JS): mount or update a card =====
window.squib.load({
  kind: "permission",    // "permission" | "plan" | "elicitation" | "default"
  tool: "bash",          // "bash" | "edit" | "read" | "write" | "glob" | "grep" | "agent" | "plan" | "ask"
  title: "Permission Request",
  folder: "self-service-frontend",
  shortId: "969",
  command: "brew update && brew upgrade",    // permission/plan body
  compact: false,                              // drop the .sq-body block
  suggestions: [
    { label: "Allow Session", kbd: "⌘⇧S", id: "allow-session" },
    { label: "Always allow `brew update:*`",  id: "allow-always" }
  ],
  elicit: {                              // only when kind === "elicitation"
    section: "Session goal",
    question: "What would you like…",
    hint: "Choose one option",
    options: [{ id, title, desc, selected }]
  }
});

// ===== inbound (Swift → JS): chat append =====
window.squib.chat.append({
  role: "assistant",                // "user" | "assistant"
  text: "Found it. ...",
  thinking: "the user reports...",    // optional preamble
  tools: [{                            // optional tool calls
    tool: "edit", arg: "src/auth/", file: "LoginShell.tsx",
    status: "ok",                    // "ok" | "err" | "running"
    summary: "+3 −2",
    body: "<diff html>"             // pre-rendered span markup
  }]
});

// ===== outbound (JS → Swift) =====
// click on .sq-btn[data-action] / .sq-sug[data-action] / .sq-cbtn[data-action]
// posts:
{ type: "action", action: "allow" }
{ type: "action", action: "allow-session" }
{ type: "submit", optionId: "bug-fixing" }       // elicitation
{ type: "chat:send", text: "yeah, patch it" }
{ type: "chat:stop" }

// transport: webkit.messageHandlers.squib.postMessage(...)
Card sizing. Card width is 340px default, 380px for elicitation (.sq-card.wide). Chat is 460×600 (.sq-card.sq-chat). Window size in Swift should track the card — don't add NSWindow padding around it; the glass card IS the window content.

07 Port checklist
1. Lift the entire :root { --sq-* } block + everything from .sq-card down to @keyframes sqin into bubbleHTML.
2. Window: NSWindow.styleMask = [.borderless], backgroundColor = .clear, isOpaque = false; NSVisualEffectView.material = .hudWindow at the back; WKWebView with setValue(false, forKey: "drawsBackground") in front.
3. Body should set background: transparent; the card provides its own.
4. Wire data-action click delegation → webkit.messageHandlers.squib.postMessage.
5. Chat tool toggle: document.addEventListener('click', e => { const t = e.target.closest('.sq-tool .h'); if (t) t.parentElement.classList.toggle('open'); }).
6. Mount window.squib.load({...}) + window.squib.chat.append({...}) as the only Swift→JS entry points; everything else is style-only.
